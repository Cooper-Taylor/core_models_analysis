## Pattern used by every notebook

1. **Setup cell** opens a `NotebookSession`:
   ```python
   from kbutillib.notebook import NotebookSession
   session = NotebookSession.for_notebook(project_name='core_models_analysis')
   ```
   This creates `notebooks/.kbcache/` (SQLite catalog + blob store) on
   first use.
2. **Re-execute the analysis** using the moved-out scripts'
   functions or by reading the cached CSV/JSON from `../results/`.
3. **Cache the heavy outputs** with `session.cache.save(name, obj,
   type_hint=...)`. Subsequent notebook runs read from cache:
   ```python
   try:
       data = session.cache.load('rxnsets_by_model')   # <— from cache
   except KeyError:
       data = compute_it()                              # <— first time
       session.cache.save('rxnsets_by_model', data, type_hint='json')
   ```
4. **Render the matching report** verbatim as a markdown cell so the
   writeup is part of the notebook.

```bash

```

## Rebuilding

`../scripts/build_notebooks.py` is the source of truth for notebooks
00–05 — edit it (not the `.ipynb` files) and re-run:

```bash
python3 ../scripts/build_notebooks.py
jupyter execute --inplace *.ipynb
```

Notebook 06 is generated by `../scripts/build_reversibility_notebook.py`
and uses the supporting modules `../scripts/reversibility_lib.py`
(parameterizable port of `Estimate_Reaction_Reversibility.py`) and
`../scripts/growth_heuristics.py` (panel-rebound + FBA driver):

```bash
python3 ../scripts/build_reversibility_notebook.py
jupyter execute --inplace 06_ReactionReversibilityHeuristics.ipynb
```

First execution loads the 56K-reaction MSDB and runs every cascade
variant — ~45s.  Subsequent executions hit `.kbcache/` and finish in
under 5s.
