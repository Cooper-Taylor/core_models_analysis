    About

    • Cascade source: ModelSEEDDatabase/Scripts/Thermodynamics/Estimate_Reaction_Reversibility.py ported into core_models_analysis/scripts/reversibility_lib.py behind a ReversibilityConfig knob-set. Defaults reproduce the upstream cascade byte-for-byte.
    • Heuristics review: Reaction_Reversibility_Heuristics_Review.md (revision 2026-06-01, §§ 2.1 / 3.1 / 3.2 / 3.3 / 3.4 / 3.5 / 3.6 / 3.7 / 3.8 / 3.9 / 3.10 + H1–H5). Each variant tag in the browser points at one section.
    • Panel: 100 descriptive growers from core_models_analysis/results/selected_ids.txt.
    • FBA reference: variants are compared to the heuristic baseline -- panel models fully rebound to the MSDB cascade's defaults. This isolates "what does the heuristic say" from "what does the on-disk model JSON say" (which differ in 98/100 panel models because the JSON bounds are template-time, not always in sync with MSDB).
    • What's not modified: anything under ModelSEEDDatabase/ or core_models_kegg2/. All variant reports live under core_models_analysis/thermo_variants/{tag}/; all rebinding happens in memory.
    • Live FBA: the Flux Sandbox + the Reaction Explorer's "per-mode panel sweep" actually run COBRA FBA via the backend (site/serve.py → scripts/growth_heuristics.py). Each call takes ~5-30 s depending on panel size.
